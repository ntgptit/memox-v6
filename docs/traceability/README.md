# Traceability

This directory makes the delivery plan auditable without widening every WBS
table. Read:

1. [`work-item-schema.md`](./work-item-schema.md) for inherited and per-item
   metadata rules.
2. [`work-item-register.md`](./work-item-register.md) for owner/source/gate
   mappings and the first-learning chain.

The WBS row remains the source for ID, title, size, dependency, and completion
boundary. The register supplies accountability and traceability metadata. A
conflict is resolved by updating both in the same documentation change.
